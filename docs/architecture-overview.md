# 架构概览

## 1. 目标定位

CollAI 旨在构建一个“持续运行的认知型 AI 代理系统”，而不仅仅是一个会写代码的聊天机器人。它强调：

- 长期记忆
- 自主规划与执行
- 自我反思与学习
- 可演化的代码与行为
- 本地优先部署

## 2. 总体结构

从项目白皮书中可以概括为以下几层：

1. 前端层：交互界面与可视化面板
2. API 层：服务接口与控制面
3. 认知层：任务调度、循环执行与认知状态
4. 记忆层：短期/长期记忆与知识组织
5. 工具层：文件操作、执行命令、网络访问等
6. 安全与治理层：权限、审计与沙箱

## 3. 核心设计思想

- 不是靠单次对话完成任务，而是通过持续运行形成累积能力
- 通过模块化状态流转实现认知循环
- 通过记忆和演化能力形成更强的长期性能
- 通过本地优先部署降低数据与隐私风险

## 4. 现阶段仓库状态

当前仓库主要是项目文档与设计沉淀，尚未正式承载完整应用代码。后续可按以下方式推进：

- 先形成稳定的目录与模块边界
- 再拆分后端服务与前端页面
- 最后补齐测试、部署和运行说明

## 5. 后续建议

建议下一阶段先把“可运行骨架”补齐，而不是一开始就追求过多功能：

- 后端：API 服务、任务管理、认知循环骨架
- 前端：基础页面、状态管理、实时交互
- 文档：架构说明、开发指南、部署流程
